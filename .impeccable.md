# Impeccable Design Context

## Design Context

### Users

Mixed casual users — couples planning a move, solo apartment planners, anyone who wants to arrange furniture in a space. No specific professional context. They want to quickly see what fits where without learning complex CAD software.

### Brand Personality

**Friendly, practical, clear.** An approachable tool that gets out of the way — function over flair. Should feel like a helpful drafting companion, not enterprise software.

### Aesthetic Direction

**"The Digital Drafting Table"** — clean & functional (Notion/Linear territory) crossed with architectural precision. Balances cold drafting precision with the warmth of a collaborative studio.

**References:** Notion, Linear (clean, content-first, subtle interactions)
**Anti-references:** Overly corporate/enterprise software (dense dashboards, Salesforce-like complexity)

### Design Principles

1. **Precision without complexity** — The UI should feel as precise as the floorplans being created (strict 4px/8px grid, monospaced numbers) without exposing that complexity to the user.
2. **Tonal layering over borders** — Define boundaries through background color shifts, not 1px borders. The UI should feel like stacked sheets of vellum, not a grid of boxes.
3. **Workspace breathing room** — The canvas takes 70%+ of the screen. Negative space is a design tool. Left-heavy composition for controls, letting the workspace breathe.
4. **Editorial warmth** — No pure black (#000000) for text. Soft, readable tones. Rounded corners (8px) to soften the technical nature. The "warm white" palette keeps things approachable.
5. **Glass & gradient for floating elements** — Floating toolbars use glassmorphism (80% opacity + 12px backdrop-blur). Primary CTAs get subtle gradients for premium depth.

### Accessibility

WCAG AA compliance. Good contrast ratios, keyboard navigation, screen reader support, visible focus indicators.

## Design Spec: The Blueprint Precision Framework

### Colors & Surface Philosophy

**The "No-Line" Rule:** Standard 1px borders are strictly prohibited for sectioning. Use background color shifts instead.

**Surface Hierarchy:**

- Level 0 (Base): `surface` (#f7f9fb)
- Level 1 (Panels): `surface-container-low` (#f2f4f6)
- Level 2 (Cards/Modules): `surface-container-highest` (#e0e3e5)
- Level 3 (Pop-overs): `surface-container-lowest` (#ffffff) — floating, "lifted" effect

**Glassmorphism:** Floating toolbars use `surface` at 80% opacity with `backdrop-blur: 12px`. Primary CTAs use subtle linear gradient from `primary` (#000000 / #131b2e) to `primary_container`.

**"Ghost Border" Fallback:** If accessibility requires a border, use `outline-variant` (#c6c6cd) at 15% opacity. Felt, not seen.

### Typography

| Role               | Font              | Usage                                                                                                     |
| ------------------ | ----------------- | --------------------------------------------------------------------------------------------------------- |
| Display & Headline | **Manrope**       | Project titles, architectural statements. Letter-spacing: -0.02em                                         |
| Title & Body       | **Inter**         | Workhorse text, floorplan metadata, UI copy                                                               |
| Technical Labels   | **Space Grotesk** | ALL dimensions, coordinates, unit measurements. Monospaced numbers prevent jumping during drag operations |

**Rule:** Never use 100% black (#000000) for text. Use `on_surface` (#191c1e).

### Elevation & Depth

Hierarchy through **tonal layering**, not shadows.

- Floating elements (context menus): shadow with 20-40px blur at 4% opacity, `on-surface` (#191c1e) base
- No drop shadows on buttons — use color contrast for "pressability"

### Components

**Buttons:**

- Primary: `primary` background, no border, `rounded-lg` (8px)
- Secondary: `secondary_fixed` background
- Tertiary: No background, `label-md` styling, `surface-variant` hover

**Inputs:** No 4-sided box. Use "bottom-line" approach or solid `surface-container-high` fill. Error state: `error` (#ba1a1a) text + 2px bottom-indicator only (never full-container red).

**Cards & Lists:** No dividers. Use `spacing-6` (1.5rem) gaps. `surface-variant` background on hover for hit area.

**Collaboration:**

- Cursors: High-chroma distinct colors outside the main palette
- Avatars: `rounded-xl` (12px), not full circles — architectural "block-plan" feel
- Item locking: `secondary_container` tint at 20% opacity + ghost border

### Icons

Line-art style with 1.5px stroke weight to match the blueprint aesthetic. No standard filled icons.

### Spacing (The "Drafting Grid")

All layouts snap to 4px/8px grid.

- Layout margins: `spacing-8` (2rem)
- Component internal padding: `spacing-3` or `spacing-4`
- Gaps between related items: `spacing-2` (0.5rem)
- List item separation: `spacing-6` (1.5rem) — no dividers
