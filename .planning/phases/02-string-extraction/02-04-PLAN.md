  <how-to-verify>
**Setup:** Start the dev server with `bun dev`. Open the app in the browser.

**Test 1 — Language Switch + Project List (I18N-02)**

1. Open the sidebar and switch to "Deutsch"
2. Verify the project list page shows all text in German
3. Check that project card timestamps say "vor X Minuten/Stunden/Tagen" (not "5m ago")

**Test 2 — Project Detail Page (I18N-02)**

1. Open a project
2. Check sidebar action labels are in German ("Teilen", "Aktualisieren", etc.)
3. Check sidebar group titles are in German
4. Open the branch dialog — check title, placeholder, buttons are German
5. Trigger a delete confirmation dialog — check it's in German

**Test 3 — Canvas (I18N-02, I18N-04)**

1. On the canvas, right-click an item — check context menu is in German
2. Check zoom control tooltips are in German (hover over buttons)
3. Check dimension labels on items show German decimal comma if decimal (e.g., "3,5 × 2 cm")
4. Check the scale bar text

**Test 4 — History Page (I18N-02, I18N-05)**

1. Click the History button on a project
2. Check column headers are in German
3. Check filter dropdowns show German labels
4. Check timestamps show "vor X Minuten" format
5. Check pagination text is in German

**Test 5 — Number Formatting (I18N-04)**

1. Find an item with a decimal price — verify comma separator in German (e.g., "€3,50" not "€3.50")
2. Check item dimensions use comma for decimals in German

**Test 6 — Plural Forms (I18N-10)**

1. With 1 item in the list — verify "1 Element" (not "1 Elemente")
2. With 2+ items — verify "2 Elemente" (not "2 Element")

**Test 7 — Form Validation (I18N-06)**

1. Try to submit a form with a required field empty
2. Check that the browser validation message appears in German (depends on `<html lang="de">`)

**Test 8 — Switch Back to English**

1. Switch back to English
2. Verify everything reverts to English correctly
3. Verify decimal dots return (3.50 not 3,50)
