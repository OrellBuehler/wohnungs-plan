```typescript
export function uploadWithProgress(
  url: string,
  formData: FormData,
  onProgress: (percent: number) => void,
  headers?: Record<string, string>
): Promise<Response>
```
- Uses `XMLHttpRequest` with `upload.onprogress` for progress events
- IMPORTANT (Pitfall 3): Set `xhr.withCredentials = true` for same-origin session cookies

**Wire upload progress (ERRH-04):**
In `src/lib/stores/project.svelte.ts`, modify `uploadFloorplan()` and `uploadItemImage()`:

1. Import `uploadWithProgress` from `$lib/utils/upload.ts`
2. Import `toast` from `svelte-sonner`
3. Import `UploadProgress` from `$lib/components/shared/UploadProgress.svelte`
4. Import `* as m` from `$lib/paraglide/messages`

For each upload function:
- Create a progress toast using `toast.custom(UploadProgress, { duration: Infinity, componentProps: { progress: 0, filename: file.name } })` and capture the `toastId`
- Replace the `fetch` call with `uploadWithProgress(url, formData, (percent) => { toast.custom(UploadProgress, { id: toastId, duration: Infinity, componentProps: { progress: percent, filename: file.name } }) })`
- Pass the same headers that `authFetch` would use (get auth token from session)
- IMPORTANT (Pitfall 4): In a `finally` block, always `toast.dismiss(toastId)`. Then show `toast.success(m.upload_success())` on success or `toast.error(m.upload_failed())` on failure.
- On failure: keep the file selected so user can retry without re-selecting (per user decision). This means don't reset the file input — the component should preserve its state on error.

Note: The `uploadFloorplan` function builds a FormData with the floorplan file. The `uploadItemImage` function takes a File. Both need the XHR replacement.
