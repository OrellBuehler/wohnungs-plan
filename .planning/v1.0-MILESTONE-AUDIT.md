### Integration Gap: ERRH-06

The `reason=auth_required` query parameter sent by `auth.svelte.ts:68` to `/api/auth/login` is discarded by the login handler (which only reads the `redirect` param). After OAuth completes, the callback redirects to `/?login=success` — the reason param never reaches `+page.svelte`. The `toast.info(m.auth_login_required())` is dead code.

**Fix:** Forward `reason` through OAuth via cookie or reconstruct in the callback URL.
